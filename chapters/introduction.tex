Most modern text editors provide code autocompletion features that allow the developer to autocomplete the code fragment they are currently typing.
This means that the developer doesn't have to memorize all the identifiers in scope but rather can rely on knowing part of them.
The autocomplete feature is also used to speed up writing as the developers doesn't have to type out all the characters when using it. \textcite{how-programmers-interact-with-code-generation-models}
The autocomplete usually works by comparing the partial identifier being written with valid indetifiers and it suggest the ones that are the most similar.
Autocomplete alogithms that use edit based distances usually work with only completing a single token at a time as predicting longer distance relations is not accurate enough.
Recently, with developments in natural language processing, there have been attempts to use deep learning models that are capable of predecting multiple tokens ahead. \textcite{automated-code-generation-using-deeplearning}
However deep learning models require a lot of training data and computing power and there have been questions whether it is ethical to use code licensed under strong copyleft licenses to train a model that generates code without any copyleft license.\footfullcite{copilot-latest-challange-to-copyleft}
However there hasn't been so much popularity in searching for an analytical solution for the problem instead of machine learning based.
